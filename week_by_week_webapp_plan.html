BLOCK A — HTML/CSS/JS basics + project scaffolding (weeks 1–3)
Wk 1
Environment setup + HTML/CSS fundamentals
Install VS Code, Git, Node.js. Complete first 3 sections of The Odin Project (HTML Foundations). Build a static mockup of the app's budget entry form in plain HTML and CSS — no functionality yet. Commit to GitHub daily, even if just one line changes. Goal: understand the DOM, box model, and basic layout.
HTML/CSSGit
Wk 2
JavaScript basics — variables, functions, DOM manipulation
Work through javascript.info chapters 1–6 (fundamentals). Make your static form interactive: when a user types a budget figure, display a running total. No frameworks yet. Understand event listeners and querySelector. Keep the app on GitHub and write a README describing what it is and what it will do.
JavaScriptGit
Wk 3
Arrays, objects, and storing budget data in JS
Model a clinical trial budget as a JavaScript object (categories, line items, amounts, actuals vs. planned). Build a form that appends line items to an array and re-renders a table. No database yet — data lives in memory. Study localStorage briefly to persist data between page refreshes. This is your app's first real data layer.
JavaScript
BLOCK B — Introduce React, restructure app (weeks 4–6)
Wk 4
React introduction — components and props
Read the official React docs (Quick Start + Thinking in React). Scaffold the project with Vite (npm create vite@latest). Convert your existing budget form to a React component. Build BudgetCategory and LineItem components. The app should render the same table as before, now in React. Goal: understand component hierarchy and data flow downward via props.
React
Wk 5
State management — useState, controlled inputs
Replace localStorage with React state. Build a fully controlled form: add a budget line item (category, description, planned amount, actual amount), display all items in a table with subtotals per category and a grand total. Add the ability to delete a line item. This week's output is the core of your app's functionality. Budget about 3 extra hours — this is the hardest week conceptually.
React
Wk 6
Styling, layout, and a basic dashboard chart
Install Tailwind CSS via the Vite plugin. Style the app professionally: clean table, form, header, color-coded variance indicators (over/under budget). Add a simple bar chart using Recharts (npm install recharts) showing planned vs. actual by category. The app should look presentable to a non-technical stakeholder by end of week.
ReactCSS
BLOCK C — Backend, data persistence, deployment (weeks 7–9)
Wk 7
Supabase setup — cloud database and auth
Create a free Supabase project. Design the database schema: budgets table, line_items table, categories. Connect your React app to Supabase using the JS client library. Replace the in-memory state with real database reads and writes. Add basic email/password authentication so data is user-specific. This is the largest conceptual jump — plan for 4–5 hours.
ReactBackend
Wk 8
Multiple budgets, CSV export, data validation
Add support for multiple trial budgets (a user can have several projects). Build a project list/selection screen. Add CSV export using the papaparse library. Add basic form validation (required fields, numeric amounts, max length). Handle loading states and errors gracefully in the UI. The app is now functionally complete for a v1.
ReactJavaScript
Wk 9
Deploy to production + write documentation
Deploy the frontend to Vercel (free tier, connects directly to GitHub). Configure Supabase environment variables. Test the app end-to-end in production. Write a thorough README: what problem it solves, who it is for, tech stack, screenshots, live link. Write a 300-word case study in a separate document explaining your design decisions. This document becomes portfolio material.

Code review, refactor, and portfolio site launch
